\subsubsection*{\doxyref{P\+H\+P}{p.}{namespace_p_h_p} version requirements}

P\+H\+P-\/\+Parser now requires \doxyref{P\+H\+P}{p.}{namespace_p_h_p} 5.\+3 or newer to run. It is however still possible to {\itshape parse} \doxyref{P\+H\+P}{p.}{namespace_p_h_p} 5.\+2 source code, while running on a newer version.

\subsubsection*{Move to namespaced names}

The library has been moved to use namespaces with the {\ttfamily \doxyref{Php\+Parser}{p.}{namespace_php_parser}} vendor prefix. However, the old names using underscores are still available as aliases, as such most code should continue running on the new version without further changes.

Old (still works, but discouraged)\+:

```php \$parser = new (new ); \$pretty\+Printer = new ; ```

New\+:

```php \$parser = new (new \doxyref{Php\+Parser}{p.}{namespace_php_parser}); \$pretty\+Printer = new ; ```

Note that the {\ttfamily P\+H\+P\+Parser} prefix was changed to {\ttfamily \doxyref{Php\+Parser}{p.}{namespace_php_parser}}. While \doxyref{P\+H\+P}{p.}{namespace_p_h_p} class names are technically case-\/insensitive, the autoloader will not be able to load {\ttfamily P\+H\+P\+Parser\textbackslash{}Parser} or other case variants.

Due to conflicts with reserved keywords, some class names now end with an underscore, e.\+g. {\ttfamily P\+H\+P\+Parser\+\_\+\+Node\+\_\+\+Stmt\+\_\+\+Class} is now {\ttfamily \doxyref{Php\+Parser}{p.}{namespace_php_parser}\textbackslash{}Node\textbackslash{}Stmt\textbackslash{}Class\+\_\+}. (But as usual, the old name is still available.)

\subsubsection*{Changes to {\ttfamily Node\+::get\+Type()}}

The {\ttfamily Node\+::get\+Type()} method continues to return names using underscores instead of namespace separators and also does not contain the trailing underscore that may be present in the class name. As such its output will not change in many cases.
