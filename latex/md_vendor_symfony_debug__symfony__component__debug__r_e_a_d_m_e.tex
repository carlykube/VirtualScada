Debug provides tools to make debugging easier.

Enabling all debug tools is as easy as calling the {\ttfamily enable()} method on the main {\ttfamily Debug} class\+:

```php use \doxyref{Symfony}{p.}{namespace_symfony};

Debug\+::enable(); ```

You can also use the tools individually\+:

```php use \doxyref{Symfony}{p.}{namespace_symfony}; use \doxyref{Symfony}{p.}{namespace_symfony};

if (\textquotesingle{}cli\textquotesingle{} !== php\+\_\+sapi\+\_\+name()) \{ ini\+\_\+set(\textquotesingle{}display\+\_\+errors\textquotesingle{}, 0); Exception\+Handler\+::register(); \} elseif (!ini\+\_\+get(\textquotesingle{}log\+\_\+errors\textquotesingle{}) $\vert$$\vert$ ini\+\_\+get(\textquotesingle{}error\+\_\+log\textquotesingle{})) \{ ini\+\_\+set(\textquotesingle{}display\+\_\+errors\textquotesingle{}, 1); \} Error\+Handler\+::register(); ```

Note that the {\ttfamily Debug\+::enable()} call also registers the debug class loader from the \doxyref{Symfony}{p.}{namespace_symfony} \doxyref{Class\+Loader}{p.}{class_class_loader} component when available.

This component can optionally take advantage of the features of the Http\+Kernel and Http\+Foundation components.
