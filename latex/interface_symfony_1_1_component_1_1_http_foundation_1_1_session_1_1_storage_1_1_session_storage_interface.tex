Regenerates id that represents this storage.

This method must invoke session\+\_\+regenerate\+\_\+id(\$destroy) unless this interface is used for a storage object designed for unit or functional testing where a real \doxyref{P\+H\+P}{p.}{namespace_p_h_p} session would interfere with testing.

Note regenerate+destroy should not clear the session data in memory only delete the session data from persistent storage.

Care\+: When regenerating the session I\+D no locking is involved in P\+H\+Ps session design. See {\tt https\+://bugs.\+php.\+net/bug.\+php?id=61470} for a discussion. So you must make sure the regenerated session is saved B\+E\+F\+O\+R\+E sending the headers with the new I\+D. Symfonys \doxyref{Http\+Kernel}{p.}{namespace_symfony_1_1_component_1_1_http_kernel} offers a listener for this. See \doxyref{Symfony}{p.}{namespace_symfony}. Otherwise session data could get lost again for concurrent requests with the new I\+D. \doxyref{One}{p.}{class_one} result could be that you get logged out after just logging in.
