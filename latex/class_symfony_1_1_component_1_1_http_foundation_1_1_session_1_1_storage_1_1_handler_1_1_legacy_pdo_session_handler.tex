\subsection{Detailed Description}
\doxyref{Session}{p.}{class_symfony_1_1_component_1_1_http_foundation_1_1_session_1_1_session} handler using a P\+D\+O connection to read and write data.

\doxyref{Session}{p.}{class_symfony_1_1_component_1_1_http_foundation_1_1_session_1_1_session} data is a binary string that can contain non-\/printable characters like the null byte. For this reason this handler base64 encodes the data to be able to save it in a character column.

This version of the \doxyref{Pdo\+Session\+Handler}{p.}{class_symfony_1_1_component_1_1_http_foundation_1_1_session_1_1_storage_1_1_handler_1_1_pdo_session_handler} does N\+O\+T implement locking. So concurrent requests to the same session can result in data loss due to race conditions.
