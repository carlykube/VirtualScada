\subsection{Detailed Description}
\doxyref{Class\+Loader}{p.}{class_composer_1_1_autoload_1_1_class_loader} implements a P\+S\+R-\/0 class loader

See {\tt https\+://github.\+com/php-\/fig/fig-\/standards/blob/master/accepted/\+P\+S\+R-\/0.\+md} \begin{DoxyVerb}$loader = new \Composer\Autoload\ClassLoader();

// register classes with namespaces
$loader->add('Symfony\Component', __DIR__.'/component');
$loader->add('Symfony',           __DIR__.'/framework');

// activate the autoloader
$loader->register();

// to enable searching the include path (eg. for PEAR packages)
$loader->setUseIncludePath(true);
\end{DoxyVerb}


In this example, if you try to use a class in the \doxyref{Symfony}{p.}{namespace_symfony} namespace or one of its children (\doxyref{Symfony}{p.}{namespace_symfony} for instance), the autoloader will first look for the class under the component/ directory, and it will then fallback to the framework/ directory if not found before giving up.

This class is loosely based on the \doxyref{Symfony}{p.}{namespace_symfony} Universal\+Class\+Loader.
