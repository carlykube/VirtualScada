16 2.2.0
17 -----
18 
19  * [DEPRECATION] Several route settings have been renamed (the old ones will be removed in 3.0):
20 
21  * The `pattern` setting for a route has been deprecated in favor of `path`
22  * The `_scheme` and `_method` requirements have been moved to the `schemes` and `methods` settings
23 
24  Before:
25 
26  ```yaml
27  article_edit:
28  pattern: /article/{id}
29  requirements: { '_method': 'POST|PUT', '_scheme': 'https', 'id': '\d+' }
30  ```
31 
32  ```xml
33  <route id="article_edit" pattern="/article/{id}">
34  <requirement key="_method">POST|PUT</requirement>
35  <requirement key="_scheme">https</requirement>
36  <requirement key="id">\d+</requirement>
37  </route>
38  ```
39 
40  ```php
41  $route = new Route();
42  $route->setPattern('/article/{id}');
43  $route->setRequirement('_method', 'POST|PUT');
44  $route->setRequirement('_scheme', 'https');
45  ```
46 
47  After:
48 
49  ```yaml
50  article_edit:
51  path: /article/{id}
52  methods: [POST, PUT]
53  schemes: https
54  requirements: { 'id': '\d+' }
55  ```
56 
57  ```xml
58  <route id="article_edit" pattern="/article/{id}" methods="POST PUT" schemes="https">
59  <requirement key="id">\d+</requirement>
60  </route>
61  ```
62 
63  ```php
64  $route = new Route();
65  $route->setPath('/article/{id}');
66  $route->setMethods(array('POST', 'PUT'));
67  $route->setSchemes('https');
68  ```
69 
70  * [BC BREAK] RouteCollection does not behave like a tree structure anymore but as
71  a flat array of Routes. So when using PHP to build the RouteCollection, you must
72  make sure to add routes to the sub-collection before adding it to the parent
73  collection (this is not relevant when using YAML or XML for Route definitions).
74 
75  Before:
76 
77  ```php
78  $rootCollection = new RouteCollection();
79  $subCollection = new RouteCollection();
80  $rootCollection->addCollection($subCollection);
81  $subCollection->add('foo', new Route('/foo'));
82  ```
83 
84  After:
85 
86  ```php
87  $rootCollection = new RouteCollection();
88  $subCollection = new RouteCollection();
89  $subCollection->add('foo', new Route('/foo'));
90  $rootCollection->addCollection($subCollection);
91  ```
92 
93  Also one must call `addCollection` from the bottom to the top hierarchy.
94  So the correct sequence is the following (and not the reverse):
95 
96  ```php
97  $childCollection->addCollection($grandchildCollection);
98  $rootCollection->addCollection($childCollection);
99  ```

101  * [DEPRECATION] The methods `RouteCollection::getParent()` and `RouteCollection::getRoot()`
102  have been deprecated and will be removed in Symfony 2.3.
103  * [BC BREAK] Misusing the `RouteCollection::addPrefix` method to add defaults, requirements
104  or options without adding a prefix is not supported anymore. So if you called `addPrefix`
105  with an empty prefix or `/` only (both have no relevance), like
106  `addPrefix('', $defaultsArray, $requirementsArray, $optionsArray)`
107  you need to use the new dedicated methods `addDefaults($defaultsArray)`,
108  `addRequirements($requirementsArray)` or `addOptions($optionsArray)` instead.
109  * [DEPRECATION] The `$options` parameter to `RouteCollection::addPrefix()` has been deprecated
110  because adding options has nothing to do with adding a path prefix. If you want to add options
111  to all child routes of a RouteCollection, you can use `addOptions()`.
112  * [DEPRECATION] The method `RouteCollection::getPrefix()` has been deprecated
113  because it suggested that all routes in the collection would have this prefix, which is
114  not necessarily true. On top of that, since there is no tree structure anymore, this method
115  is also useless. Don't worry about performance, prefix optimization for matching is still done
116  in the dumper, which was also improved in 2.2.0 to find even more grouping possibilities.
117  * [DEPRECATION] `RouteCollection::addCollection(RouteCollection $collection)` should now only be
118  used with a single parameter. The other params `$prefix`, `$default`, `$requirements` and `$options`
119  will still work, but have been deprecated. The `addPrefix` method should be used for this
120  use-case instead.
121  Before: `$parentCollection->addCollection($collection, '/prefix', array(...), array(...))`
122  After:
123  ```php
124  $collection->addPrefix('/prefix', array(...), array(...));
125  $parentCollection->addCollection($collection);
126  ```
127  * added support for the method default argument values when defining a @Route
128  * Adjacent placeholders without separator work now, e.g. `/{x}{y}{z}.{_format}`.
129  * Characters that function as separator between placeholders are now whitelisted
130  to fix routes with normal text around a variable, e.g. `/prefix{var}suffix`.
131  * [BC BREAK] The default requirement of a variable has been changed slightly.
132  Previously it disallowed the previous and the next char around a variable. Now
133  it disallows the slash (`/`) and the next char. Using the previous char added
134  no value and was problematic because the route `/index.{_format}` would be
135  matched by `/index.ht/ml`.
136  * The default requirement now uses possessive quantifiers when possible which
137  improves matching performance by up to 20% because it prevents backtracking
138  when it's not needed.
139  * The ConfigurableRequirementsInterface can now also be used to disable the requirements
140  check on URL generation completely by calling `setStrictRequirements(null)`. It
141  improves performance in production environment as you should know that params always
142  pass the requirements (otherwise it would break your link anyway).
143  * There is no restriction on the route name anymore. So non-alphanumeric characters
144  are now also allowed.
145  * [BC BREAK] `RouteCompilerInterface::compile(Route $route)` was made static
146  (only relevant if you implemented your own RouteCompiler).
147  * Added possibility to generate relative paths and network paths in the UrlGenerator, e.g.
148  "../parent-file" and "//example.com/dir/file". The third parameter in
149  `UrlGeneratorInterface::generate($name, $parameters = array(), $referenceType = self::ABSOLUTE_PATH)`
150  now accepts more values and you should use the constants defined in `UrlGeneratorInterface` for
151  claritiy. The old method calls with a Boolean parameter will continue to work because they
152  equal the signature using the constants.

154 2.1.0
155 -----
156 
157  * added RequestMatcherInterface
158  * added RequestContext::fromRequest()
159  * the UrlMatcher does not throw a \LogicException anymore when the required
160  scheme is not the current one
161  * added TraceableUrlMatcher
162  * added the possibility to define options, default values and requirements
163  for placeholders in prefix, including imported routes
164  * added RouterInterface::getRouteCollection
165  * [BC BREAK] the UrlMatcher urldecodes the route parameters only once, they
166  were decoded twice before. Note that the `urldecode()` calls have been
167  changed for a single `rawurldecode()` in order to support `+` for input
168  paths.
169  * added RouteCollection::getRoot method to retrieve the root of a
170  RouteCollection tree
171  * [BC BREAK] made RouteCollection::setParent private which could not have
172  been used anyway without creating inconsistencies
173  * [BC BREAK] RouteCollection::remove also removes a route from parent
174  collections (not only from its children)
175  * added ConfigurableRequirementsInterface that allows to disable exceptions
176  (and generate empty URLs instead) when generating a route with an invalid
177  parameter value
