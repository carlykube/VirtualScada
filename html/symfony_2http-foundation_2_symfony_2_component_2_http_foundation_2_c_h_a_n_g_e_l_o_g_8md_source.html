4 2.6.0
5 -----
6 
7  * PdoSessionHandler changes
8  - implemented different session locking strategies to prevent loss of data by concurrent access to the same session
9  - [BC BREAK] save session data in a binary column without base64_encode
10  - [BC BREAK] added lifetime column to the session table which allows to have different lifetimes for each session
11  - implemented lazy connections that are only opened when a session is used by either passing a dsn string
12  explicitly or falling back to session.save_path ini setting
13  - added a createTable method that initializes a correctly defined table depending on the database vendor

15 2.5.0
16 -----
17 
18  * added `JsonResponse::setEncodingOptions()` & `JsonResponse::getEncodingOptions()` for easier manipulation
19  of the options used while encoding data to JSON format.

21 2.4.0
22 -----
23 
24  * added RequestStack
25  * added Request::getEncodings()
26  * added accessors methods to session handlers

28 2.3.0
29 -----
30 
31  * added support for ranges of IPs in trusted proxies
32  * `UploadedFile::isValid` now returns false if the file was not uploaded via HTTP (in a non-test mode)
33  * Improved error-handling of `\Symfony\Component\HttpFoundation\Session\Storage\Handler\PdoSessionHandler`
34  to ensure the supplied PDO handler throws Exceptions on error (as the class expects). Added related test cases
35  to verify that Exceptions are properly thrown when the PDO queries fail.

37 2.2.0
38 -----
39 
40  * fixed the Request::create() precedence (URI information always take precedence now)
41  * added Request::getTrustedProxies()
42  * deprecated Request::isProxyTrusted()
43  * [BC BREAK] JsonResponse does not turn a top level empty array to an object anymore, use an ArrayObject to enforce objects
44  * added a IpUtils class to check if an IP belongs to a CIDR
45  * added Request::getRealMethod() to get the "real" HTTP method (getMethod() returns the "intended" HTTP method)
46  * disabled _method request parameter support by default (call Request::enableHttpMethodParameterOverride() to
47  enable it, and Request::getHttpMethodParameterOverride() to check if it is supported)
48  * Request::splitHttpAcceptHeader() method is deprecated and will be removed in 2.3
49  * Deprecated Flashbag::count() and \Countable interface, will be removed in 2.3

51 2.1.0
52 -----
53 
54  * added Request::getSchemeAndHttpHost() and Request::getUserInfo()
55  * added a fluent interface to the Response class
56  * added Request::isProxyTrusted()
57  * added JsonResponse
58  * added a getTargetUrl method to RedirectResponse
59  * added support for streamed responses
60  * made Response::prepare() method the place to enforce HTTP specification
61  * [BC BREAK] moved management of the locale from the Session class to the Request class
62  * added a generic access to the PHP built-in filter mechanism: ParameterBag::filter()
63  * made FileBinaryMimeTypeGuesser command configurable
64  * added Request::getUser() and Request::getPassword()
65  * added support for the PATCH method in Request
66  * removed the ContentTypeMimeTypeGuesser class as it is deprecated and never used on PHP 5.3
67  * added ResponseHeaderBag::makeDisposition() (implements RFC 6266)
68  * made mimetype to extension conversion configurable
69  * [BC BREAK] Moved all session related classes and interfaces into own namespace, as
70  `Symfony\Component\HttpFoundation\Session` and renamed classes accordingly.
71  Session handlers are located in the subnamespace `Symfony\Component\HttpFoundation\Session\Handler`.
72  * SessionHandlers must implement `\SessionHandlerInterface` or extend from the
73  `Symfony\Component\HttpFoundation\Storage\Handler\NativeSessionHandler` base class.
74  * Added internal storage driver proxy mechanism for forward compatibility with
75  PHP 5.4 `\SessionHandler` class.
76  * Added session handlers for custom Memcache, Memcached and Null session save handlers.
77  * [BC BREAK] Removed `NativeSessionStorage` and replaced with `NativeFileSessionHandler`.
78  * [BC BREAK] `SessionStorageInterface` methods removed: `write()`, `read()` and
79  `remove()`. Added `getBag()`, `registerBag()`. The `NativeSessionStorage` class
80  is a mediator for the session storage internals including the session handlers
81  which do the real work of participating in the internal PHP session workflow.
82  * [BC BREAK] Introduced mock implementations of `SessionStorage` to enable unit
83  and functional testing without starting real PHP sessions. Removed
84  `ArraySessionStorage`, and replaced with `MockArraySessionStorage` for unit
85  tests; removed `FilesystemSessionStorage`, and replaced with`MockFileSessionStorage`
86  for functional tests. These do not interact with global session ini
87  configuration values, session functions or `$_SESSION` superglobal. This means
88  they can be configured directly allowing multiple instances to work without
89  conflicting in the same PHP process.
90  * [BC BREAK] Removed the `close()` method from the `Session` class, as this is
91  now redundant.
92  * Deprecated the following methods from the Session class: `setFlash()`, `setFlashes()`
93  `getFlash()`, `hasFlash()`, and `removeFlash()`. Use `getFlashBag()` instead
94  which returns a `FlashBagInterface`.
95  * `Session->clear()` now only clears session attributes as before it cleared
96  flash messages and attributes. `Session->getFlashBag()->all()` clears flashes now.
97  * Session data is now managed by `SessionBagInterface` to better encapsulate
98  session data.
99  * Refactored session attribute and flash messages system to their own
100  `SessionBagInterface` implementations.
101  * Added `FlashBag`. Flashes expire when retrieved by `get()` or `all()`. This
102  implementation is ESI compatible.
103  * Added `AutoExpireFlashBag` (default) to replicate Symfony 2.0.x auto expire
104  behaviour of messages auto expiring after one page page load. Messages must
105  be retrieved by `get()` or `all()`.
106  * Added `Symfony\Component\HttpFoundation\Attribute\AttributeBag` to replicate
107  attributes storage behaviour from 2.0.x (default).
108  * Added `Symfony\Component\HttpFoundation\Attribute\NamespacedAttributeBag` for
109  namespace session attributes.
110  * Flash API can stores messages in an array so there may be multiple messages
111  per flash type. The old `Session` class API remains without BC break as it
112  will allow single messages as before.
113  * Added basic session meta-data to the session to record session create time,
114  last updated time, and the lifetime of the session cookie that was provided
115  to the client.
116  * Request::getClientIp() method doesn't take a parameter anymore but bases
117  itself on the trustProxy parameter.
118  * Added isMethod() to Request object.
119  * [BC BREAK] The methods `getPathInfo()`, `getBaseUrl()` and `getBasePath()` of
120  a `Request` now all return a raw value (vs a urldecoded value before). Any call
121  to one of these methods must be checked and wrapped in a `rawurldecode()` if
122  needed.
