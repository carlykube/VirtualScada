8 ## About
9 
10 PsySH is a runtime developer console, interactive debugger and [REPL](http://en.wikipedia.org/wiki/Read%E2%80%93eval%E2%80%93print_loop) for PHP. Learn more at [psysh.org](http://psysh.org/). Check out the [Interactive Debugging in PHP talk from OSCON](https://presentate.com/bobthecow/talks/php-for-pirates) on Presentate.

13 ## Installation
14 
15 Download the `psysh` phar to install:
16 
17 ```
18 wget psysh.org/psysh
19 chmod +x psysh
20 ./psysh
21 ```
22 
23 It's even awesomer if you put it somewhere in your system path (like `/usr/local/bin` or `~/bin`)!
24 
25 PsySH [is available via Composer](https://packagist.org/packages/psy/psysh), so you can use it in your project as well:
26 
27 ```
28 composer require psy/psysh:@stable
29 ./vendor/bin/psysh
30 ```
31 
32 Or you can use by checking out the the repository directly:
33 
34 ```
35 git clone https://github.com/bobthecow/psysh.git
36 cd psysh
37 ./bin/psysh
38 ```

41 ## PsySH configuration
42 
43 While PsySH strives to detect the right settings automatically, you might want to configure it yourself. Just add a file to `~/.config/psysh/config.php` (or `C:\Users\{USER}\AppData\Roaming\PsySH` on Windows):
44 
45 ```php
46 <?php
47 
48 return array(
49  // In PHP 5.4+, PsySH will default to your `cli.pager` ini setting. If this
50  // is not set, it falls back to `less`. It is recommended that you set up
51  // `cli.pager` in your `php.ini` with your preferred output pager.
52  //
53  // If you are running PHP 5.3, or if you want to use a different pager only
54  // for Psy shell sessions, you can override it here.
55  'pager' => 'more',
56 
57  // Sets the maximum number of entries the history can contain.
58  // If set to zero, the history size is unlimited.
59  'historySize' => 0,
60 
61  // If set to true, the history will not keep duplicate entries.
62  // Newest entries override oldest.
63  // This is the equivalent of the HISTCONTROL=erasedups setting in bash.
64  'eraseDuplicates' => false,
65 
66  // By default, PsySH will use a 'forking' execution loop if pcntl is
67  // installed. This is by far the best way to use it, but you can override
68  // the default by explicitly enabling or disabling this functionality here.
69  'usePcntl' => false,
70 
71  // PsySH uses readline if you have it installed, because interactive input
72  // is pretty awful without it. But you can explicitly disable it if you hate
73  // yourself or something.
74  'useReadline' => false,
75 
76  // PsySH automatically inserts semicolons at the end of input if a statement
77  // is missing one. To disable this, set `requireSemicolons` to true.
78  'requireSemicolons' => false,
79 
80  // "Default includes" will be included once at the beginning of every PsySH
81  // session. This is a good place to add autoloaders for your favorite
82  // libraries.
83  'defaultIncludes' => array(
84  __DIR__.'/include/bootstrap.php',
85  ),
86 
87  // While PsySH ships with a bunch of great commands, it's possible to add
88  // your own for even more awesome. Any Psy command added here will be
89  // available in your Psy shell sessions.
90  'commands' => array(
91  // The `parse` command is a command used in the development of PsySH.
92  // Given a string of PHP code, it pretty-prints the
93  // [PHP Parser](https://github.com/nikic/PHP-Parser) parse tree. It
94  // prolly won't be super useful for most of you, but it's there if you
95  // want to play :)
96  new \Psy\Command\ParseCommand,
97  ),
98 
99  // PsySH ships with presenters for scalars, resources, arrays, and objects.
100  // But you're not limited to those presenters. You can enable additional
101  // presenters (like the included MongoCursorPresenter), or write your own!
102  'presenters' => array(
103  new \Psy\Presenter\MongoCursorPresenter,
104  ),
105 
106  // You can disable tab completion if you want to. Not sure why you'd want to.
107  'tabCompletion' => false,
108 
109  // You can write your own tab completion matchers, too! Here are some that enable
110  // tab completion for MongoDB database and collection names:
111  'tabCompletionMatchers' => array(
112  new \Psy\TabCompletion\Matcher\MongoClientMatcher,
113  new \Psy\TabCompletion\Matcher\MongoDatabaseMatcher,
114  ),
115 );
116 ```
